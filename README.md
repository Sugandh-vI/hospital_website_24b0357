## Brief

This is the last assignment of the LS, and it's deliberately the most open-ended one. No fixed API shape, no required endpoints, no theme to pick from a list — just a real brief: **build a website for a hospital.**

Everything you've learned across all four weeks — HTML/CSS/JS fundamentals, React, Django + DRF + PostgreSQL, auth, and now deployment and AI-assisted workflow — comes together here. Use the [AI-Assisted Development workflow](../02-AI-Assisted-Development/README.md) from this week to plan it properly before you touch code, and the [Deployment guide](../01-Deployment/README.md) to actually ship it.

## Requirements

There's no fixed list of endpoints or pages this time — that's intentional. At minimum, think through what a hospital website realistically needs (a home page, information about doctors/departments, some way to see or book appointments, contact/emergency info are reasonable starting points, not a checklist), then decide what's crucial, what's a nice-to-have, and what's out of scope, the same way you'd plan any real project.

**The grading is purely based on:**
1. **Features** — what the site actually does, and how complete/thoughtful it is
2. **UI** — visual design and polish
3. **User Experience** — how easy and intuitive it is for someone unfamiliar with the project to actually use it

If you've built a feature that's genuinely useful but easy to miss or stumble across, add a short section on the site describing it.

## What to Submit

Your submission is:
1. The **live website URL** (from your Week 4 Deployment work) with your **frontend folder renamed to your roll number** so it's clearly visible above the URL when someone opens the website.

This naming is **crucial** — it's how submissions get verified as your own project.

---

**Rules:**
- React (Vite) for the frontend, Django + DRF for the backend — same stack as the rest of the course.
- Deploy it. A submission that only runs on `localhost` cannot be graded on UI/UX properly — follow the [Deployment guide](../01-Deployment/README.md) from this week.
- Beyond the frontend-folder naming rule above, the rest of the structure and feature set is your call.
